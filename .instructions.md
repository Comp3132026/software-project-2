# LifeSync AI Ruleset

## Project Scope
LifeSync is a **group task collaboration app** with real-time chat, notifications, and role-based member management. Tech stack: MERN (MongoDB, Express, React, Node).

## Frontend (React + Tailwind)
- Use `authAPI`, `groupsAPI`, `tasksAPI`, `membersAPI`, `chatAPI`, `notificationsAPI` from `utils/api.js`
- Handle all API errors with `toast.error(err.response?.data?.message || 'Failed')`
- Fetch data in `useEffect` with try-catch
- Validate form inputs before submit
- Use custom hooks: `useGroupData`, `useGroupActions`, `useMemberActions`, `useTaskAction`
- Context: `AuthContext`, `GroupContext`, `NotificationContext`
- Color scheme: Tailwind (primary-600, surface-100, dark-800, etc.)

## Backend (Express + Mongoose)
- All routes protected with `auth` middleware from `middleware/auth`
- Pattern: Validate input → Check permission → Execute → Return JSON response
- Error responses: `{ message: "...", error: "..." }` with appropriate status codes
- Models: User, Group, Task, Message, Notification, Warning
- Relationships: Groups have members (User refs), Tasks assigned to users, Messages in groups
- No delete operations - use soft delete where needed (isDeleted flag + timestamp)

## RBAC (4 Tiers)
- **Owner**: Full control of group
- **Moderator**: Manage tasks, members, warnings
- **Member**: Create tasks in group
- **Viewer**: Read-only access

Check permissions in routes before sensitive operations.

## Validation
- Frontend: Validate before form submit
- Backend: Use `express-validator` on route handlers, check field lengths and formats
- Email: Must be valid email format
- Password: Min 6 characters
- Name: Min 2 characters

## Must DO
✅ Use auth middleware on all protected routes
✅ Populate user/group refs when querying
✅ Return consistent error format
✅ Validate userId/groupId ObjectId format
✅ Use toast.error/success for user feedback
✅ JSDoc comments on complex functions

## Must NOT
❌ Expose user _id without authorization check
❌ Allow deletion without permission check
❌ Skip error handling in async routes
❌ Hardcode API URLs (use VITE_API_BASE_URL)
❌ Trust user ID from request body (use req.userId from token)
❌ Create permanent deletes (use soft delete instead)
